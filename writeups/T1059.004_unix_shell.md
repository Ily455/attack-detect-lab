# T1059.004 — Command and Scripting Interpreter: Unix Shell

## Technique

**MITRE ATT&CK:** [T1059.004](https://attack.mitre.org/techniques/T1059/004/)  
**Tactic:** Execution  
**Platform:** Linux

Adversaries abuse Unix shell interpreters (bash, sh, dash) to execute commands, run scripts, and chain operations. It is one of the most fundamental execution techniques on Linux — nearly every post-exploitation action involves a shell at some level.

---

## Test Environment

- **Target:** Ubuntu 22.04 ARM64 (UTM VM)
- **Attacker:** macOS M3 triggering via SSH
- **Tool:** Atomic Red Team — `Invoke-AtomicTest T1059.004`
- **Logging:** auditd with execve syscall rules, shipped via Filebeat to Elasticsearch

## Execution

```bash
./run-test.sh T1059.004
```

Atomic ran 17 sub-tests covering:
- Bash script creation and execution
- Shell enumeration (`/etc/shells`, `$SHELL`)
- Obfuscated commands via base64
- SUID file harvesting (missing prereqs — skipped)
- Shell spawning via awk, cpan, busybox
- Environment variable abuse
- Kernel information enumeration

**Result:** 15/17 tests succeeded. 2 failed: one due to missing prerequisite files (AutoSUID.sh, LinEnum.sh not downloaded), one due to missing emacs binary.

## Evidence

auditd captured every shell invocation via the `execve` syscall. Example from test 1:

```
type=EXECVE msg=audit(1780489845.066:1912): argc=3 a0="sh" a1="-c" a2=""
```

Key fields:
- `type=EXECVE` — syscall captured
- `a0` — executable (`sh`, `bash`, `awk`, etc.)
- `a1` — flag (`-c` = command string follows)
- `a2` — the command that was run

---

## Detection

### KQL (Kibana)

```
message: "type=EXECVE" AND (message: "a0=\"sh\"" OR message: "a0=\"bash\"" OR message: "a0=\"dash\"") AND message: "a1=\"-c\""
```

See [`detections/T1059.004_unix_shell.yml`](../detections/T1059.004_unix_shell.yml)

## Notes

- Shell invocations with `-c` are extremely common in legitimate system activity — this rule will generate false positives in production and requires tuning
- The key signal is context: who spawned the shell, from what parent process, at what time
- Enriching with `ppid` (parent PID) from auditd `SYSCALL` records improves precision significantly
